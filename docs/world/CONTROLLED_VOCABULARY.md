# Registro de vocabulários controlados

## Objetivo

Este documento centraliza propriedade, responsabilidade, estado e política de extensão dos vocabulários usados pelos contratos editoriais. Ele é um índice normativo: não substitui as listas e regras completas dos schemas responsáveis e não cria conteúdo de Eldrath.

## Regras de governança

- A hierarquia documental do projeto continua determinando precedência.
- Um campo terminado em `_key` usa uma chave ASCII em `snake_case`; uma referência de entidade usa ID completo.
- Lista mínima expressamente aprovada pode ser estendida somente pelo documento proprietário, com justificativa, compatibilidade e revisão.
- Valor renomeado ou removido exige migração documentada; significado existente não pode ser alterado silenciosamente.
- Quando a lista final depende de conteúdo ou implementação futura, o registro informa proprietário, formato e fase, sem inventar enum definitivo.
- `null` representa valor aplicável ainda desconhecido; não autoriza uma chave fora do vocabulário.

## Critério de uso

Um autor pode usar imediatamente somente os valores aprovados acima ou os valores já definidos pelo schema responsável. Para qualquer registro `adiado explicitamente`, a estrutura do campo está contratada, mas o valor concreto deve aguardar a fase e a tarefa responsáveis. A ausência de lista final não autoriza texto livre, chave improvisada nem valor definitivo criado para preencher um template.
